第五屆 WeHelp Bootcamp 計畫申請

兩百字內的個人簡介。

嗨，我是阿平，大學主修資訊管理。興趣廣泛、個性認真，覺得有趣的事就會認真探索研究。

因為覺得「能救人的資料科學很性感」而參加 D4SG 的活動，跟團隊分析高雄急診室資料並建立預測心肺休止的模型，而後參加過多場黑客松比賽皆有獲得名次。

畢業後曾在台大跟高醫當研究助理，也在中經院當過工程師。（詳見個人履歷）

現在的我在過運動員生活，跟台灣最頂尖的長跑運動員一起訓練，希望自己能成為一個心技體都足夠強韌的人。

為了成為軟體工程師，做過什麼努力？若有具體作品請分享給我們。

雖然身為資訊相關科系畢業生，但求學時碰的技術比較少，當台大研究助理時，我曾參加過彭彭老師的台大資訊系統訓練班（前端工程入門），算是開啟我學習網頁前端的契機。

我認為資訊能力應該是用來解決實際的問題，所以我大部分都是透過參加黑客松/專案來學習，或著透過網路資源（ex: hahow）自學，但缺點就是黑客松這種短期專案通常都是以解決問題為主（能動就好）、且當時只有自己會寫程式只能自己摸索，沒人一起討論。「缺乏系統性的學習與討論交流」是最大的問題，這也是我思考許久後，最後決定申請 WeHelp 的原因。

以下是我閒暇時自主開發的小作品，都是為了解決我自己生活中會遇到的問題，前兩個是純前端 html + css + javascript，第三個是利用 flask + heroku 串 line 機器人的嘗試。

    • 如果參與這個訓練，會怎麼安排學習時間？

    我的技術底子並不好，雖不是完全沒基礎，但知道自己距離真正的軟體工程師還有好大一段距離，所以我是抱著重新歸零的心態來參與這個訓練營。

    我的生活本來就很單純，如果有幸入選，除了維持原本早晨的跑步訓練外，其餘時間我會用來全職參與這個訓練計畫。

    跟練馬拉松一樣，講求的是耐心跟持續累積：前期我會確保自己在過程中找到堅持下去的樂趣，而不是一開始就使勁衝結果力竭，導致無法堅持到最後。

      • 12:30-13:30 午餐

      • 19:00-21:00 寫作業(前期)、進行開發(後期)

      • 另一個半天則是複習一週以來的學習進度，濃縮精煉，產出一週的學習成果報告。

    身為運動愛好者，會想加入類似運動筆記、跑步筆記、健行筆記之類的公司，想接觸跟跑步／運動科學／運動員／爬山等相關題目的開發。

    請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

    前面提到「台北馬拉松配速計算機」的專案，是一個完全不會寫程式的朋友找我開發的，當時想用自己的專業為跑步圈貢獻，也想累積作品於是無償答應下來，並開放原始碼。然而因為某些緣故鬧得不愉快，導致朋友後來又找更專業的人做了功能一模一樣的東西、開放營利，還在網站中的介紹文字狠狠酸我。

    這件事情傷害我非常大，被拋棄、自我否定...等負面情緒接踵而來，然而當時自己正在準備年底的半馬比賽，情緒不能受影響，於是我告訴自己「先處理事情再處理情緒」，我知道自己有情緒，但我也知道面對負面採取的行為是自己可以選擇的。

    我反問自己「你希望因為這件事當作你沒跑好的藉口嗎？」，得到了否定的答案，既然我如此重視那場比賽，那我就要做對得起自己的決定，不要被擊垮。最後於該場21公里的半馬比賽中，我跑出了1:21:37，在全部16,689名參賽者中，總名次第112名(99.33%)。

    這次的挫敗經驗，也是我非常強烈想要參加 WeHelp Bootcamp 的原因，我知道自己無須向對方證明甚麼，也無意繼續吵架謾罵，但從哪裡跌倒就從哪裡站起來，我不缺創新想法跟題目，但想增進自己在資訊能力的專業，未來才不容易被取代，才能做出更有感、能觸動人心的作品。
